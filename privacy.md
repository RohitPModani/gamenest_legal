## Introduction

Welcome to **Listist**. We respect your privacy and are committed to protecting your personal information. This Privacy Policy explains how we collect, use, store, and protect your information when you use the Listist mobile application ("App," "Service," "we," "us," or "our").

By using Listist, you agree to the collection and use of information in accordance with this Privacy Policy.

## 1. Information We Collect

### 1.1 Information You Provide Directly

**Account Information (When Using Google Sign-In):**
- Email address
- Full name
- Profile picture/avatar URL
- Google user ID (for authentication)

**User-Generated Content:**
- List titles and notes (max 50 characters for titles, 200 for notes)
- List items with titles, URLs, and notes (max 50 characters for titles, 300 for notes)
- Group/folder names and descriptions (max 30 characters)
- Category selections (from 16 predefined categories)
- Item completion status (checked/unchecked)
- Collaboration invitations (email addresses of invitees)
- Friend list (email addresses you've invited)

**Anonymous Usage:**
- If you use the app without signing in, we generate a unique anonymous ID stored locally on your device
- Anonymous data is stored in our database with this ID instead of a user account

### 1.2 Information Collected Automatically

**Device Information:**
- Device model and manufacturer
- Operating system version (iOS/Android)
- App version number
- Device language and region settings

**Usage Data:**
- App launch times and session duration
- Features used (list creation, collaboration, etc.)
- Navigation patterns within the app
- Error logs and crash reports
- Performance metrics

**Network Information:**
- IP address (for authentication and security)
- Connection type (Wi-Fi, cellular)
- Connectivity status (online/offline detection)

### 1.3 Information from Third-Party Services

**Google Sign-In:**
- Basic profile information (name, email, profile picture)
- Google user ID for authentication
- OAuth tokens (stored securely, never exposed)

**Supabase (Backend):**
- Database connection metadata
- Query performance metrics
- Authentication session data

**Firebase Services:**
- Crash reports and stack traces
- Analytics events (app opens, feature usage)
- Performance monitoring data
- Device and app information

## 4. Data Storage and Security

### 4.1 Where We Store Data

**Supabase (Primary Database):**
- PostgreSQL database hosted on secure cloud infrastructure
- Data centers located in [Specify Region if known, or "multiple regions worldwide"]
- Compliant with industry security standards

**Local Device Storage:**
- Anonymous user ID (if using anonymously)
- App preferences and settings
- Cached data for offline access

### 4.2 Security Measures

We implement industry-standard security practices:

**Encryption:**
- All data transmitted via HTTPS/TLS encryption
- Data encrypted at rest in the database
- OAuth tokens securely managed

**Access Controls:**
- Row-Level Security (RLS) in database
- User authentication required for most features
- Role-based permissions (owner vs. member)

**Authentication:**
- Google OAuth 2.0 for secure sign-in
- Session tokens with automatic expiration
- Secure token storage

**Infrastructure Security:**
- Regular security updates
- Automated backups
- Intrusion detection
- Firewall protection

**Application Security:**
- Input validation and sanitization
- Protection against SQL injection
- CSRF protection
- Secure coding practices

### 4.3 Data Retention

**Active Accounts:**
- Data retained indefinitely while account is active
- You can delete data at any time

**Deleted Accounts:**
- Account data deleted within 30 days
- Backup systems may retain data up to 30 additional days
- Anonymized analytics may be retained longer

**Anonymous Users:**
- Data retained until you clear app data or delete the app
- Can be migrated to authenticated account

**Collaboration Data:**
- Deleted lists are removed for all members
- Leaving a list removes your access but not the list itself
